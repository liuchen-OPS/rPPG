# 科学级rPPG心率检测系统设计方案

## 项目概述

本项目是一个基于Web的远程光电容积描记（rPPG）心率检测系统，利用普通摄像头实现非接触式心率、心率变异性（HRV）、血氧饱和度（SpO2）等生理指标的实时监测。

---

## 核心功能

### 1. 实时心率检测
- **检测范围**: 40-220 bpm
- **精度**: ±2-3 bpm（静息状态）
- **更新频率**: 每秒1次
- **算法**: FFT频谱分析 + 抛物线插值

### 2. 心率变异性（HRV）检测
- **指标**: SDNN（标准差）
- **评估**: 优秀(>100ms) / 正常(50-100ms) / 偏低(<50ms)
- **临床意义**: 自主神经系统功能评估

### 3. 血氧饱和度估算（SpO2）
- **范围**: 70-100%
- **说明**: 基于RGB比值的近似估算，仅供参考
- **准确性**: 低于医疗级设备，不能用于诊断

### 4. 面部距离检测
- **检测范围**: 20-150 cm
- **最佳距离**: 30-60 cm
- **功能**: 辅助用户调整检测位置

### 5. 五分钟综合报告
- **检测时长**: 5分钟（医学标准）
- **自动结束**: 报告生成后自动停止
- **报告内容**: 8项生理指标综合分析

### 核心算法

#### 1. 人脸检测与ROI定位
```
MediaPipe Face Mesh (468关键点)
    ↓
额头ROI计算（基于关键点坐标）
    ↓
坐标缩放映射（适应显示区域）
```

#### 2. 信号预处理流程
```
原始RGB信号
    ↓
异常值去除（3-4σ准则）
    ↓
多项式去趋势（3-5阶）
    ↓
Butterworth带通滤波（0.7-4.5Hz）
    ↓
陷波滤波（50Hz电源干扰）
    ↓
运动伪影去除（中值滤波）
    ↓
小波去噪（软阈值）
    ↓
归一化
```

#### 3. 心率计算流程
```
预处理后的信号
    ↓
FFT频谱分析（1024点）
    ↓
峰值检测（7点窗口 + SNR阈值）
    ↓
抛物线插值精细化
    ↓
谐波验证
    ↓
心率平滑（MAD滤波 + 自适应EMA）
    ↓
变化率限制（最大5bpm/次）
```

#### 4. 运动后检测优化
- **自动识别**: 信号方差>2.0或心率>100bpm
- **扩展频率范围**: 0.8-4.5Hz（48-270bpm）
- **放宽异常值阈值**: 4σ（vs 3σ静息）
- **增强去噪**: 更严格的滤波参数

```

    ↓

    • 显示信号质量指标
    • 收集数据用于报告

    ↓

    ↓

```

## 数据收集与报告

### 收集指标
| 指标 | 频率 | 用途 |
|------|------|------|
| 心率 | 1Hz | 平均值、范围、稳定性 |
| HRV | 每次心跳 | 平均SDNN |
| SpO2 | 1Hz | 平均值 |
| 面部距离 | 1Hz | 平均距离 |
| SNR | 1Hz | 平均信噪比、信号质量评级 |

### 报告结论生成逻辑
```
1. 心率评估
   - 60-100bpm: 正常静息范围
   - >100bpm: 偏高，可能处于活动状态
   - <60bpm: 偏低，可能是运动员体质

2. 心率稳定性
   - <3bpm: 非常稳定
   - 3-6bpm: 较为稳定
   - >6bpm: 波动较大

3. HRV评估
   - >100ms: 优秀，自主神经功能健康
   - 50-100ms: 正常
   - <50ms: 偏低，可能疲劳或压力

4. 信号质量
   - SNR≥10dB: 良好，结果可信
   - 5-10dB: 一般，可供参考
   - <5dB: 较差，建议重新测量
```

## 性能优化

### 1. 算法优化
- **减少计算量**: 移除Savitzky-Golay滤波等复杂计算
- **简化平滑算法**: 使用本地循环替代方法调用
- **优化峰值检测**: 减少排序操作，使用近似算法

### 2. 内存管理
- **缓冲区限制**: 信号缓冲区固定长度（300点）
- **历史数据限制**: 心率历史最多10个值
- **定时清理**: 停止检测时清空所有缓冲区

### 3. 日志优化
- **减少输出**: 仅保留关键事件日志
- **避免循环日志**: 不在每帧输出调试信息

## 使用建议

### 最佳检测条件
1. **光照**: 均匀、充足的自然光或室内光
2. **距离**: 30-60cm，面部占画面1/3-1/2
3. **姿势**: 正对摄像头，保持静止
4. **背景**: 纯色、简洁，避免复杂图案
5. **时间**: 静息5分钟后检测，避免运动后立即检测

### 运动后检测
- **等待时间**: 运动后休息30-60秒
- **指标变化**: 心率偏高、HRV暂时降低是正常的
- **血氧参考**: 运动后血氧显示"参考"，准确性降低

### 注意事项
- ⚠️ 本系统仅供健康监测参考，不能替代医疗设备
- ⚠️ 血氧估算值误差较大，不能用于医疗诊断
- ⚠️ 检测过程中保持面部稳定，避免说话、大笑
- ⚠️ 5分钟检测期间不要离开摄像头视野

## 技术限制

### 无法实现的功能
| 功能 | 原因 |
|------|------|
| 血压检测 | 需要压力传感器，光学成像无法测量 |
| 精确血氧 | 需要红外光，普通摄像头只有可见光 |
| 医疗级精度 | 消费级摄像头和算法的固有限制 |

### 精度限制
| 指标 | 预期精度 | 说明 |
|------|----------|------|
| 心率 | ±2-3 bpm | 静息状态，良好光照 |
| HRV | ±10-20% | 基于心率计算，间接测量 |
| SpO2 | ±5-10% | 估算值，仅供参考 |
| 面部距离 | ±5-10 cm | 基于面部宽度的几何估算 |

## 总结

本系统是一个基于Web的科学级rPPG心率检测工具，通过5分钟的标准检测流程，提供心率、HRV、血氧等生理指标的综合评估。虽然受限于消费级硬件，但在良好条件下可以提供有价值的健康参考数据。

**核心优势**:
- ✅ 非接触式检测，使用方便
- ✅ 5分钟医学标准检测时长
- ✅ 自动报告生成，结果直观
- ✅ 支持运动后检测

**使用限制**:
- ⚠️ 不能替代医疗设备
- ⚠️ 需要保持良好检测条件
- ⚠️ 血氧等指标仅供参考
