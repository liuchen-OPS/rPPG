# 心率检测安卓应用

这是一个基于rPPG技术的心率检测安卓应用，可以将Python心率检测代码转换为安卓APK可执行文件。

## 文件结构

```
rPPG安卓化/
├── heart_rate_precision_only.py     # 原始Python心率检测代码
├── heart_rate_android_kivy.py       # Kivy兼容的安卓版本
├── main.py                          # 简化版安卓应用主文件
├── buildozer.spec                   # Buildozer构建配置文件
├── build_apk.bat                    # Windows构建脚本
└── README.md                        # 说明文档
```

## 系统要求

### 开发环境要求
- Windows 10/11
- Python 3.7+
- Java Development Kit (JDK) 8+
- Android SDK
- Android NDK

### 目标设备要求
- Android 5.0+ (API 21+)
- 前置摄像头
- 至少1GB RAM

## 构建步骤

### 方法1: 使用构建脚本（推荐）

1. **双击运行构建脚本**
   ```
   双击运行 build_apk.bat
   ```

2. **等待构建完成**
   - 首次构建需要下载Android SDK/NDK，可能需要较长时间
   - 构建过程中请保持网络连接稳定

3. **获取APK文件**
   - 构建成功后，APK文件位于：`bin/heartrateapp-0.1-debug.apk`

### 方法2: 手动构建

1. **安装必要工具**
   ```bash
   pip install buildozer kivy numpy
   ```

2. **初始化Buildozer**
   ```bash
   buildozer init
   ```

3. **构建APK**
   ```bash
   buildozer -v android debug
   ```

## 应用功能

### 主要功能
- ✅ 实时心率检测
- ✅ 摄像头预览
- ✅ 信号质量评估
- ✅ 心率状态显示
- ✅ 开始/停止检测控制

### 技术特性
- 基于rPPG（远程光电容积描记）技术
- 使用绿色通道信号进行心率检测
- FFT频率分析算法
- 实时信号质量评估
- 安卓兼容的Kivy界面

## 使用说明

### 安装APK
1. 将APK文件传输到安卓设备
2. 在设备上启用"未知来源"安装权限
3. 安装APK文件

### 使用应用
1. **启动应用**：点击应用图标启动
2. **开始检测**：点击"开始检测"按钮
3. **对准摄像头**：将面部对准前置摄像头
4. **保持稳定**：保持面部稳定以获得准确结果
5. **查看结果**：实时显示心率值和信号质量
6. **停止检测**：点击"停止检测"按钮

### 最佳实践
- 在光线充足的环境下使用
- 保持面部正对摄像头
- 避免剧烈运动或说话
- 连续检测至少10秒以获得稳定结果

## 技术实现

### 信号处理流程
```
摄像头帧 → 绿色通道提取 → 信号缓存 → 去趋势处理 → 
移动平均滤波 → FFT频率分析 → 心率计算 → 结果显示
```

### 安卓兼容性优化
- 使用Kivy Camera替代OpenCV
- 降低分辨率提高性能（320x240）
- 简化算法减少计算负载
- 适配安卓权限系统

## 常见问题

### Q: 构建过程中出现网络错误？
A: 首次构建需要下载大量依赖，请确保网络稳定，或使用代理。

### Q: APK安装失败？
A: 请检查设备是否允许安装未知来源应用。

### Q: 心率检测不准确？
A: 确保光线充足，面部稳定，避免环境干扰。

### Q: 应用闪退？
A: 检查设备是否满足系统要求，特别是摄像头权限。

## 开发说明

### 代码结构
- `main.py`: 应用主入口，包含Kivy界面和核心逻辑
- `buildozer.spec`: 构建配置文件
- `heart_rate_android_kivy.py`: 完整功能版本（可选）

### 自定义修改
如需修改应用功能，可以编辑以下文件：

1. **修改界面**：编辑`main.py`中的`build()`方法
2. **调整算法**：修改信号处理相关函数
3. **更改配置**：更新`buildozer.spec`中的参数

### 重新构建
修改代码后，重新运行构建脚本即可生成新版APK。
